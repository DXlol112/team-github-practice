### Вопросы
1. Просто папка с историей и проектом, для совместной работы которая находится на удалёном сервере.
2. Local на лич.компьютере, а удалённый на стороннем и позволяет другим пользователям присокденится.
3. Скачиват и обновляет изменения из удалённого репозитория.
4. Отпровяет изменения в репозитории позволяя остельным участникам их применить.
5. fetch скачивает изменения и позволяет их посмотреть, а pull сразу применяет их после скачивания
6. Независимая линия разработки, которая позволяет проверить доработку в коде не затрагивая основной код.
7. Любая ошибка в коде сразу попадет в рабочую версию проекта.
8. Предложения внести изменения из одной ветки в другую.
9. Помогает искать ошибки.
10. Конфликт когда Git не может объеденить две версии в одну из за того что в одной и той же строке били внесены разные изменения.
11. Потому что два человека внесли разные изменения в одной и тоёже строке в одном файле(или кто то его удалил).
12. Выбрать тот который более актуален и не ломает код.
13. Вы будете работать не в актуальной версии проекта и при её загрузке будет ошибка.
14. Проще отменить небольшое действие и исправить его чем отенять большой кусок изменённого кода и переписывать его заного.
15. Коммуникаци с участниками.
